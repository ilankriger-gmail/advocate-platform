# Criar documentacao de arquitetura com diagramas de fluxo de dados

## Overview

O projeto utiliza uma arquitetura complexa com Server Components, Client Components, Server Actions, Supabase RLS, e multiplos modulos (auth, posts, challenges, events, rewards). Nao existe documentacao visual ou textual explicando como esses componentes se relacionam.

## Rationale

Sem documentacao de arquitetura, novos desenvolvedores precisam ler todo o codigo para entender o fluxo de dados. Isso aumenta drasticamente o tempo de onboarding e risco de bugs arquiteturais.
